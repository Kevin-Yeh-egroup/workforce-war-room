# 工讀生人力戰情系統規格 v0.1

## 目標

把自訂義名詞審核、工讀生工作進度、人才庫與派工建議串成同一套可追蹤系統。第一版先採「報告優先、Kevin 確認後寫入」：每週五 13:30 自動產出審核清單，確認後才進行詞庫新增、審核成功、關閉事件與建立薪資。

## 每週流程

1. 讀取馴錢師星展專案中最新審核狀態為進行中的事件。
2. 找出評論標題符合「完成自定義名詞修正耗時 X 分鐘」的評論。
3. 解析候選詞、核定分鐘數、工讀生與事件資料。
4. 依 Kevin 的詞庫規則判斷 `可納入` 或 `不納入`。
5. 先產生確認清單，不直接寫入外部系統。
6. Kevin 確認後才新增詞庫、審核成功、關閉事件、建立薪資。
7. 將完成案件回流到人才庫：工作量、品質訊號、常見錯誤、適合任務與下次派工建議。

## 自動化邊界

- 不自動新增 Kevin 尚未確認的詞庫。
- 不自動處理低信心、可能有正常語意、人名、單字、標點或一般彈性詞。
- 不自動建立薪資例外規則。
- 不自動給工讀生負面評價，只記錄可觀察品質訊號與派工風險。
- 登入、Profile、組織、分鐘數、成員或薪資範本不清楚時停止並回報。

## 第一版資料欄位

### 人才庫

- 姓名
- InfoCenter 成員 ID
- 薪資範本
- 可承接工作
- 本週完成案數
- 本週核定分鐘數
- 品質穩定度
- 常見錯誤類型
- 目前負載
- 下次派工建議
- 注意事項

### 案件進度

- 事件 ID
- 事件標題
- 工讀生
- 審核狀態
- 核定分鐘數
- 詞庫新增狀態
- 審核成功狀態
- 事件關閉狀態
- 薪資建立狀態
- 卡關原因

### 詞庫判斷

- 錯誤文字
- 正確文字
- 判斷結果
- 原因
- 是否已存在詞庫
- Kevin 是否確認
- 寫入結果

## Dashboard v0.1

`index.html` 是第一版靜態戰情系統，先用 seed data 顯示：

- 本週待審核與完成狀況
- 工讀生負載與品質訊號
- 自訂義名詞候選詞判斷
- 薪資預估
- 下週派工建議

後續可把 seed data 改成由每週 automation 匯出的 JSON。
